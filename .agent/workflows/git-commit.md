# 项目开发规范

1. **语言与框架**：

- 始终使用 React + JSX。
- React 组件必须使用 Functional Components 和 Hooks。

2. **包管理**：

- 必须使用 `pnpm` 安装依赖。不要使用 `npm` 或 `yarn`。

3. **提交规范**：

- Git commit message 必须使用中文。
- 格式遵循：`<type>(<scope>): <subject>` (例如: `feat(auth): 添加登录功能`)。
- 生成的提交信息句末不要加句号。

4. **修改规范**：

- 禁止修改现有的变量名,函数名,组件名,类名,接口名,文件名等。

5. **代码量与任务拆解**：

- 生成的文件内容尽量不要超过 500 行。
- 进行开发任务拆解时，拆解的隔离度要细，防止上下文太长，导致执行过程中卡死。
